# Installation

Release: `50.5.9-prod-r22.4-preflight-contract-hotfix`
## Requirements

- Debian 12+ or Ubuntu 22.04+
- systemd
- root access
- 4 GiB RAM recommended; more for large history and many concurrent users
- local SSD/NVMe recommended
- outbound access to GitHub, Docker registry and package repositories
- domain mode additionally requires public DNS plus inbound TCP 80/443

The installer installs Docker, PostgreSQL 17 + TimescaleDB, Python venv, Gunicorn, systemd units, retention/backup timers, Nginx/Certbot in domain mode, and the management CLI.

## Public IP mode

```bash
curl -fsSL \
https://raw.githubusercontent.com/tuanchu1121/virtinfra-monitor/main/install.sh \
| bash -s -- \
--public-ip 203.0.113.10 \
--port 8080
```

Optional fixed secrets without putting them directly in shell history:

```bash
read -rsp 'Admin password: ' BW_ADMIN_PASSWORD
echo
read -rsp 'Agent token: ' BW_MONITOR_TOKEN
echo
export BW_ADMIN_PASSWORD BW_MONITOR_TOKEN

curl -fsSL \
https://raw.githubusercontent.com/tuanchu1121/virtinfra-monitor/main/install.sh \
| bash -s -- \
--public-ip 203.0.113.10 \
--port 8080

unset BW_ADMIN_PASSWORD BW_MONITOR_TOKEN
```

The token may be left empty and generated by the installer.

## Domain mode

```bash
curl -fsSL \
https://raw.githubusercontent.com/tuanchu1121/virtinfra-monitor/main/install.sh \
| bash -s -- \
--domain monitor.example.com \
--email ops@example.com
```

## Optional firewall

Only enable automated UFW after confirming the real SSH port:

```bash
curl -fsSL \
https://raw.githubusercontent.com/tuanchu1121/virtinfra-monitor/main/install.sh \
| bash -s -- \
--domain monitor.example.com \
--email ops@example.com \
--firewall \
--ssh-port 1812
```

## Optional Redis page cache

```bash
... install.sh ... --redis-cache
```

Redis is disabled by default and is never authoritative storage.

## Post-install

```bash
virtinfra-monitorctl status
virtinfra-monitorctl doctor
virtinfra-monitorctl db-check
virtinfra-monitorctl urls
virtinfra-monitorctl credentials
```

Do not expose TCP 55432. PostgreSQL is intentionally bound to loopback only.
